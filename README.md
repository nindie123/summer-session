# summer-session
小学期项目协作
## 项目预期架构
### 系统架构

整体分层流程：**临床设备层 → 数据采集标准化层 → 实时计算异常检测层 → LLM Agent 智能诊断层 → 可视化交互层**

#### 1. 临床设备层
系统原始数据来源，接入全院床边监护设备
- 设备：监护仪、呼吸机、输液泵、血气分析仪、体温探头
- 传输协议：HL7、DICOM、设备私有协议

#### 2. 数据采集与标准化层
统一治理多源异构医疗设备数据
- 多协议适配解析
- 患者与设备绑定（床号、腕带、EMR 关联）
- 数据脱敏、合规处理
- Kafka 实时消息队列数据分发

#### 3. 实时计算与异常检测层
大数据实时流式分析与风险计算
- 基于 Flink 实现体征趋势、变化率、均值实时计算
- 自动计算 MEWS/NEWS/PEWS 危重症评分
- 基线偏离、生理模式异常检测
- 时序数据存储：HBase 实时存储、HDFS 归档存储

#### 4. LLM Agent 智能诊断层
系统 AI 推理核心，实现临床级综合研判
##### 数据支撑
- RAG 医疗知识库：疾病知识、药品信息、设备参数阈值
- 临床指南库：诊疗规范、急救流程、护理常规
- 患者历史库：既往病史、过敏史、检验记录

### 核心能力
- 多参数趋势综合解读（摒弃单点阈值报警）
- 病情分级：稳定 / 预警 / 危急 / 濒死
- 智能鉴别诊断、检查与治疗建议输出
- 病情恶化预测性预警
- 所有 AI 结论支持临床指南溯源

#### 5. 可视化与交互层
多终端临床应用入口
- 护士站大屏：病区患者风险热力图展示
- 医生工作站：体征时间轴 + AI 诊断建议
- 移动端 APP：危急值推送、语音播报、一键处置
- 管理驾驶舱：科室质控、告警响应效率统计

#### 架构优势
- 分层解耦，模块清晰，易于迭代扩展
- 全链路实时流式数据处理
- LLM 多参数融合研判，大幅降低传统监护误报率
- 智能结果可溯源，符合医疗合规要求
